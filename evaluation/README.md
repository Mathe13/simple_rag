# Evaluation Module

This directory contains tools for evaluating the simple RAG system's performance and answer quality.

## 1. Load Testing with Locust

Locust is used to simulate concurrent users chatting with the system to measure response times and identify bottlenecks. The tests simulate realistic user interactions using the `qa_dataset.json` file.

**How to run Load Tests:**

First, ensure the entire application stack is running via Docker Compose (`docker compose up -d`).

Then, from the root of the project, start locust:
```bash
uv run locust -f evaluation/locustfile.py
```

Once running, open your browser to `http://localhost:8089` to start the test and view the metrics dashboard.

## 2. Accuracy Benchmarking with Ragas

Ragas is used to evaluate the quality of the answers generated by the local Llama model and the relevancy of the documents retrieved by PGVector. It uses standard metrics like Faithfulness, Answer Relevancy, Context Precision, and Context Recall.

**How to run Ragas Benchmarking:**

Ensure the local `model-server` and `postgres` database are running since the benchmark hits the local LLMs and vector database directly.

Run the script from the root directory:
```bash
uv run python evaluation/benchmark_ragas.py
```

**Parameters:**
- `--limit`: (Optional) Limit the number of questions to evaluate from the dataset to speed up testing.

*Example (Evaluate only the first 5 questions):*
```bash
uv run python evaluation/benchmark_ragas.py --limit 5
```

The script outputs a summary in the console and saves a detailed CSV report (`rag_benchmark_report.csv`) in the evaluation directory.
